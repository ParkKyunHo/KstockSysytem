## §0. 디자인 철학

### 0.1 5대 원칙

```yaml
원칙 1: 차트 없음
  HTS가 차트를 제공함
  사용자는 차트는 HTS에서 봄
  웹 대시보드는 "관제 센터" 역할
  → 텍스트, 숫자, 테이블 중심

원칙 2: 단순함 우선 (헌법 5)
  필요한 정보만 표시
  과도한 시각화 회피
  명확한 계층 구조
  
원칙 3: 빠른 의사결정
  핵심 정보 즉시 파악
  3초 이내 상태 인식
  주요 액션 1~2 클릭 도달

원칙 4: 안전성 강조
  중요 액션은 명확한 확인 절차
  손절/매도 등 위험 작업 강조
  되돌릴 수 없는 액션 경고

원칙 5: 일관성
  같은 의미는 같은 색
  같은 액션은 같은 위치
  표준 패턴 반복
```

### 0.2 사용 시나리오

```yaml
주요 사용 패턴:
  1. 아침 09:00: 전체 추적 종목 점검
  2. 장중: 알림 받고 빠르게 확인
  3. 점심: 박스 설정 추가
  4. 장 마감 후: 일일 마감 검토
  5. 주말: 추적 종목 리뷰

핵심 페이지 사용 빈도:
  대시보드: 매일 수시로 (가장 많이)
  포지션 모니터: 보유 시 자주
  추적 종목 관리: 1일 1회
  박스 설정: 신규 종목 추가 시 (주 2~3회)
  리포트: 월 1~2회 (필요 시)
  알림 센터: 알림 확인 시
  설정: 거의 안 봄
```

### 0.3 대상 사용자

```yaml
사용자: 박균호 (8년 전업 트레이더)
  - 데스크톱 메인 (모니터 다수)
  - 모바일 보조 (이동 중)
  - HTS와 병행 사용
  - 키보드 단축키 환영
  - 정보 밀도 높게 선호 (전문가)
  - 화려한 애니메이션 비선호
```

## §1. 디자인 시스템

### 1.1 기술 스택

```yaml
프레임워크: React 18+
스타일: Tailwind CSS v3+
컴포넌트: shadcn/ui
상태 관리: 
  - 서버 상태: TanStack Query (React Query)
  - 클라이언트 상태: Zustand 또는 useState
라우팅: React Router v6
폼: React Hook Form + Zod
차트: 사용 안 함 (필요 최소 시 Recharts)
```

### 1.2 색상 팔레트

```yaml
의미별 색상 (필수 일관):

상태 색상:
  TRACKING: text-slate-500 (회색, 중립)
  BOX_SET: text-blue-500 (파란, 대기)
  POSITION_OPEN: text-emerald-500 (초록, 활성)
  POSITION_PARTIAL: text-cyan-500 (청록, 부분)
  EXITED: text-zinc-400 (회색, 종료)

손익 색상 (한국식):
  수익 (양수): text-red-500 (빨강 ★ 주의)
  손실 (음수): text-blue-500 (파랑 ★ 주의)
  중립 (0): text-slate-500

중요: 한국 주식 시장은 빨강=상승, 파랑=하락
서양 표준과 반대. 헷갈리지 말 것.

알림 등급:
  CRITICAL: text-red-600 bg-red-50 (빨강)
  HIGH: text-orange-500 bg-orange-50 (주황)
  MEDIUM: text-yellow-600 bg-yellow-50 (노랑)
  LOW: text-blue-500 bg-blue-50 (파랑)

브랜드 색상:
  Primary: indigo-600 (메인 액션)
  Secondary: slate-600 (보조 액션)
  Destructive: red-600 (위험 액션)
  Success: emerald-600 (긍정 확인)

배경:
  Light: bg-white, bg-slate-50
  Dark: bg-slate-900, bg-slate-800
```

### 1.3 타이포그래피

```yaml
폰트 패밀리:
  본문: 'Pretendard', 'Inter', system-ui
  숫자: 'JetBrains Mono', 'Roboto Mono'
  
  ※ 숫자는 모노스페이스 (정렬 + 가독성)

크기:
  text-xs: 메타 정보, 시간
  text-sm: 본문 (기본)
  text-base: 강조 본문
  text-lg: 카드 제목
  text-xl: 섹션 제목
  text-2xl: 페이지 제목
  text-3xl: 핵심 숫자 (손익 등)

굵기:
  font-normal: 본문
  font-medium: 강조
  font-semibold: 제목
  font-bold: 핵심 숫자

숫자 포맷:
  가격: 73,500 (천 단위 콤마, 원 단위)
  비율: +5.23% (소수점 둘째 자리)
  수량: 100주 (단위 표시)
  손익: +245,000원 (부호 표시)
```

### 1.4 간격 (Spacing)

```yaml
스케일 (Tailwind 기본):
  p-1 = 4px, p-2 = 8px, p-3 = 12px, p-4 = 16px
  p-6 = 24px, p-8 = 32px
  
컨테이너:
  카드 안 패딩: p-4 또는 p-6
  카드 간 간격: gap-4
  섹션 간 간격: gap-8

폼 요소:
  레이블 ↔ 입력: mt-1.5
  필드 간격: space-y-4
  버튼 그룹: gap-2
```

## §3. 화면 1: 로그인 + 2FA

### 3.1 로그인 페이지 (/login)

```yaml
레이아웃: 중앙 정렬, 카드형

구조:
  ┌────────────────────────────┐
  │                            │
  │     V7.1 로고              │
  │     K-Stock Trading        │
  │                            │
  │  ┌──────────────────────┐  │
  │  │ 사용자명             │  │
  │  │ [입력 필드]          │  │
  │  │                      │  │
  │  │ 비밀번호             │  │
  │  │ [입력 필드, 마스킹]  │  │
  │  │                      │  │
  │  │ [로그인] (Primary)   │  │
  │  └──────────────────────┘  │
  │                            │
  │  © 2026                    │
  └────────────────────────────┘

크기:
  카드: max-w-sm (~384px)
  중앙 정렬

검증 (클라이언트):
  사용자명: 3자 이상, 영숫자_
  비밀번호: 8자 이상

상태:
  로딩 중: 버튼 비활성 + 스피너
  실패: Toast (빨강) + 입력 필드 에러
  성공: TOTP 페이지로 이동 또는 대시보드

에러 메시지:
  - "사용자명 또는 비밀번호가 올바르지 않습니다"
  - "잠시 후 다시 시도해주세요" (Rate Limit)

키보드:
  Enter: 로그인 시도
  Tab: 다음 필드
```

### 3.2 TOTP 입력 페이지 (/login/totp)

```yaml
구조:
  ┌────────────────────────────┐
  │                            │
  │   2단계 인증              │
  │                            │
  │   Google Authenticator의   │
  │   6자리 코드를 입력하세요  │
  │                            │
  │  ┌─────┬─────┬─────┐     │
  │  │  3  │  7  │  4  │ ... │
  │  └─────┴─────┴─────┘     │
  │  [확인]                    │
  │                            │
  │  코드를 못 받으셨나요?     │
  │  백업 코드 사용            │
  └────────────────────────────┘

특징:
  - 6개 분리된 입력 필드 (UX 좋음)
  - 자동 포커스 이동
  - 자동 제출 (6자리 입력 완료 시)
  - 30초 타이머 표시 (TOTP 갱신 주기)

세션 만료:
  로그인 페이지 1단계부터 다시
  "세션이 만료되었습니다" Toast

대안:
  - 백업 코드 입력 페이지
  - 비밀번호 재설정 (관리자 연락 안내)
```

### 3.3 2FA 최초 설정 (/settings/totp/setup)

```yaml
첫 사용 시 강제 또는 설정에서 활성화:

Step 1: QR 코드 표시

  │ Google Authenticator로     │
  │ 아래 QR 코드를 스캔하세요  │
  │                            │

  │                            │
  │ 또는 수동 입력:            │
  │   BASE32...XXX             │
  │                            │
  │ [다음]                     │
  └────────────────────────────┘

Step 2: 코드 검증
  6자리 입력 → 검증 성공 → 활성화

Step 3: 백업 코드 표시
  ┌────────────────────────────┐
  │ ⚠️ 백업 코드 (10개)        │
  │ 안전한 곳에 보관하세요     │
  │                            │
  │  1234-5678                 │
  │  9012-3456                 │
  │  ...                       │
  │                            │
  │ [복사] [다운로드]          │
  │ [완료] (체크박스 확인 후)  │
  └────────────────────────────┘

  체크박스: "백업 코드를 안전한 곳에 보관했습니다"
```

## §4. 화면 2: 대시보드 (메인)

### 4.1 레이아웃

```yaml
경로: /dashboard
역할: 한눈에 시스템 + 거래 상태 파악

구성 (위에서 아래):
  1. 상단 KPI 카드 4개
  2. 시스템 상태 + 시장 상태 (한 줄)
  3. 진입 임박 박스 (있으면 강조)
```

### 12.1 폼 검증

```yaml
실시간 검증:

  - 필드 아래 빨간 텍스트 (text-sm text-red-500)

에러 메시지:
  명확하고 구체적
  나쁜 예: "Invalid"
  좋은 예: "박스 상단은 하단보다 커야 합니다 (현재: 73000 ≤ 74000)"

  - 필드 보더 초록 (선택)
```

```yaml
페이지 로딩:
  Skeleton 컴포넌트 (shadcn/ui)

  버튼 비활성 + 스피너 (Loader2 아이콘 회전)

  Progress 바
```

```yaml
shadcn/ui Sonner 사용
```
